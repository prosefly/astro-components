## Start With The Reader's Task

This package is for Astro projects that publish MDX documentation. It gives
authors a small set of components for the patterns readers expect in docs:
installation choices, warnings, linked next steps, project trees, and short
procedures.

<Callout type="tip" title="Using Lotus?">
  Lotus already configures the markdown transforms and icon preloading. In most
  Lotus projects, authors only import the components they need in each MDX
  page.
</Callout>

<CardGrid>
  <Card icon="lucide:rocket" title="Add it to Astro" href="/docs/installation/">
    Install the package, import components in MDX, and optionally enable icons
    or markdown transforms in `astro.config.ts`.
  </Card>
  <Card icon="lucide:blocks" title="Choose a component" href="/docs/overview/">
    Match each reader need to the right component: callout, tab, accordion,
    card, step list, file tree, badge, or icon.
  </Card>
  <Card icon="lucide:terminal-square" title="Show install choices" href="/docs/package-manager-tabs/">
    Convert package-manager commands into synced tabs without making authors
    write every variant by hand.
  </Card>
  <Card icon="lucide:image" title="Improve screenshots" href="/docs/images/">
    Turn image-only Markdown paragraphs into figures and galleries.
  </Card>
  <Card icon="lucide:palette" title="Match your site" href="/docs/styling/">
    Use `--pl-*` CSS custom properties to align the components with your docs
    shell or design system.
  </Card>
</CardGrid>
